java集合总结

本文字数： 6.5k 阅读时长 ≈ 6 分钟
有关java集合的继承关系图以及集合中的重点方法和属性。

Java集合的继承关系图

集合可以分成以单个方式存储元素的Collection接口类和以键值对方式存储的Map接口类。

Collection 接口的继承关系图

Collection 接口下的所有集合都实现了Iterator接口，可以迭代！

其中List有序可重复，Set无序不可重复！

List集合的实现及重要方法

Set 集合的实现和主要方法类似！

new一个Set集合对象实际上是new一个Map集合对象，将Set集合放在Map键值对的Key区域，所以key也是无序不可重复的

Map 接口的继承关系图

Map接口存储元素是以键值对（Key）和（Value）的方式，key部分的元素无序不可重复，起决定性作用！

  • HashMap 的实现

作为key部分的元素必须重写equals和hashCode方法

  • 底层是哈希表/散列表的数据结构：数组与单向链表的结合体，充分发挥二者的优点

    • 实际上哈希表是一个一位数组，数组的每个元素是一个单向链表。

    • 有个内部静态类Node<K,V>，内含hash值：key的hashCode（）方法执行的结果，可作为下标

  • 两个重要的方法的实现原理

    • map.put（k,v）实现原理

      • 首先将k，v封装都Node对象中

      • 底层调用K的hashCode（）得到的哈希值

      • 通过哈希函数或者哈希算法，将hash值转换成数组的下标，下标位置上若没有任何元素就把Node添加到这个位置上；若该下标位置有链表，此时会拿着k和链表上的每个节点的k进行equals，若都是false则添加到末尾；否则若有一个equals返回了true，则替换这个节点中的Value值。
        哈希表结构

    • map.get（k）方法实现原理

      • 先调用k的hashCode（）方法得到哈希值，通过哈希算法转换成数组下标，通过下标定位到某个位置，若该位置什么也没有，返回null，若有单向链表，拿着k和链表中的每个k进行equals，若全是false，则返回false；否则，返回equals为true的value值。

    • 所以放在HashMap集合key部分的元素（即放在HashSet集合中）需要同时重写euqals和hashCode方法！

    • HashMap允许key和value为null；但为空的值同样也只能有一个。

    • HashTable不允许key和value的值为null；它的方法都带有synchronized关键字，线程安全

    • HashTable的初始化容量为11；默认加载因子也是0.75；扩容是原容量*2+1。

    • Property的key和value都是String类型的，属性类对象；

    • 两个方法

      • 存值：setProperty（key,value）；

      • 取值（通过key获取value）：getProperty（key）;

  • 同时重写hashCode和equals方法

    • k部分元素的特点 ：无序不可重复

  • 初始化容量必须是2的倍数，这是因为达到上列均匀，为了提高HashMap集合的存取效率

  • HashMap集合的初始化容量为16，默认加载因子为0.75；即当底层数组容量达到75%之后开始扩容。

  • 如果一个类的equals方法重写了，那么hashCode方法也必须重写

    • hashCode方法和equals方法直接使用IDEA工具生成且必须同时生成！

  • JDK8之后若HashMap的哈希一个单向链表的数量多于8个时，单向链表会会变成红黑树的数据结构，若红黑树上的元素少于6个变化回单向链表。（提高检索效率）。

  • TreeMap 以及 TreeSet 的实现

TreeMap集合底层是一个二叉树,TreeSet中的元素：无序不可重复；但是可以按照元素的大小顺序自动排序,可排序集合（升序）.

但是自定义的类型排序默认无法直接比较！

有两种方法实现自定义 类型的比较大小：Comparable和Comparator两种方法实现自定义类排序！

  • ①必须自己实现comparable接口才能比较

    • 自定义类型实现Comparable接口并实现其中的compareTo方法。

    • 在compareTo方法中指定比较规则c2.compareTo(c1);//this是c2；

    • 比较规则基本不变

  • ②实现比较器接口Comparator的方法进行比较（在new一个TreeSet类的时候给一个Comparator参数实现构造器）

    • 重写compare方法（与compareTo方法相似）

    • 符合OCP原则

    • 比较规则会变化时使用

自平衡二叉树遵循左小右大原则存放

遍历方法：

- 前序：根左右

- 中序：左根右

- 后序：左右根

TreeSet/TreeMap采用：中序遍历方式；Iterator迭代器采用中序遍历；

Collections 工具类

位于java.util包下

①、List集合从线程不安全变成线程安全：Collections.synchronizedList(Arraylist);

②、排序（对可排序的集合元素 排序，eg：String,Integer）需要保证List集合中的元素实现了Comparable接口Collections.sort(ArrayList);

​ Collections.sort(ArrayList，Comparator);
